Memories are Markdown files that an agent can keep beyond one conversation.

Scope controls who can use a memory: global user memory is shared, agent-user memory is about the user for one agent, and agent memory belongs to one agent.

Pinned memories are loaded at the start of every applicable run and therefore use context tokens. Keep them short and reserve them for information that should always apply.

Unpinned memories remain available in the agent's memory directories and can be read with filesystem tools when needed. Created and updated timestamps are generated automatically.

Conversation messages are not memory records. They remain in their threads and can be searched separately.
