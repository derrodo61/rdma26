Prices are stored per one million tokens. A token is a small piece of text sent to or generated by a model. The provider reports the token counts after each request.

Input / 1M
The price for normal input tokens sent to the model, including instructions, conversation messages, memory, and tool descriptions.
Cached input / 1M
Input tokens from an unchanged beginning of a recent request, such as repeated system instructions, tool definitions, or conversation history. The provider can reuse its earlier processing of this text and charges a lower price. Cached tokens are part of the reported input total, not additional tokens.
Output / 1M
The price for tokens generated by the model, including the visible answer and any billable reasoning tokens reported as output.
Status
Only active records are used. When a record is inactive, usage is still recorded but its monetary cost is shown as unpriced.
Source
The provider page from which the saved prices were retrieved.
Updated
When this local pricing record was last changed.

Example calculation

Suppose a request reports 100,000 input tokens, including 40,000 cached tokens, plus 10,000 output tokens. Using GPT-5.4 prices:

60,000 normal input × $2.50 / 1M = $0.15
40,000 cached input × $0.25 / 1M = $0.01
10,000 output × $15.00 / 1M = $0.15
Estimated total = $0.31

Estimates depend on provider-reported usage and the active local price. They may differ from the final provider invoice, especially for long-context or special pricing tiers.
